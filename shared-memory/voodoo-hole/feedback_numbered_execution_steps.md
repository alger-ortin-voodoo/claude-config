When presenting sequential execution steps / roadmaps, label them with numbers (Step 1, 2, 3),
not letters (Build A, B, C).

**Why:** During the Collections Probability Sets planning session (2026-06-04), the user misread
"Build A / Build B" as two *alternative* implementation approaches rather than sequential phases —
letters signal options, numbers signal order.

**How to apply:** Use numbers for anything sequential (phases, builds, commits, roadmap rows).
Reserve letters for genuinely parallel alternatives the user must choose between.
